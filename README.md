# 出纳管理系统（供应商账务）

一个纯前端、零后端的供应商付款管理工具，使用 IndexedDB 本地存储数据，支持 GitHub Pages 一键部署。

## ✨ 功能特性
- 供应商信息管理（名称、供应产品、付款类型、入库金额、实付金额）
- 多账户支持（对公/私用账户自由添加）
- 付款记录与确认（未确认→已确认，确认后自动更新实付金额）
- 按日/月/年查询任意供应商或全部供应商的付款情况
- 数据可视化图表展示
- 所有数据保存在浏览器本地，无需服务器

## 🚀 部署到 GitHub Pages

1. **创建仓库**  
   在 GitHub 上新建一个公开仓库（例如 `cashier-system`）。

2. **上传文件**  
   将以下三个文件上传到仓库根目录：
   - `index.html`
   - `config.js`
   - `README.md`（可选）

3. **启用 Pages**  
   进入仓库 `Settings` → `Pages` → 将分支设为 `main`（或 `master`），文件夹选 `/ (root)`，保存。

4. **访问系统**  
   几分钟后，通过 `https://你的用户名.github.io/仓库名/` 访问即可。

## 🔐 默认登录凭据
- 用户名：`admin`
- 密码：`admin123`

**修改方法**：编辑 `config.js` 中的 `USERNAME` 和 `PASSWORD` 值，重新上传即可生效。

## 📝 使用指南
- **仪表板**：查看供应商总数、入库/实付/未付款金额，以及待确认付款列表。
- **供应商管理**：增删改供应商，为每个供应商添加多个银行账户（对公/私用）。
- **付款记录**：新增付款记录（默认为“待确认”），确认后实付金额自动累加。
- **付款统计**：选择日期维度（日/月/年）和供应商，查询付款总额并展示图表。

## ⚠️ 注意事项
- 系统使用浏览器 **IndexedDB** 存储数据，**清除浏览器缓存会导致数据丢失**，请定期备份（可自行扩展导出功能）。
- 所有数据仅存储在当前使用的设备及浏览器中，不同设备之间不共享。

## 📄 技术栈
- HTML5 / CSS3 (TailwindCSS)
- JavaScript (ES6)
- IndexedDB
- Chart.js

## 🛠️ 本地测试
直接在浏览器中打开 `index.html` 即可运行（需确保 `config.js` 在同一目录）。
